# Vehicles

Ein Fahrzeug repräsentiert ein spezifisches EV (elektrisches Fahrzeug) und seine Batterie. Wenn ein Fahrzeug konfiguriert und einem [Ladepunkt](loadpoints#vehicle--vehicles) zugewiesen ist, kann in der Benutzeroberfläche der Ladestatus (SOC) und die verbleibende Ladezeit angezeigt werden.

Ebenfalls ist es damit möglich die Ladung auf einen bestimmten Ladestand (SoC) zu begrenzen. Da die meisten Wallboxen diesen nicht kennen können (er wird nur in bestimmten Kombinationen der Wallbox übermittelt), kann evcc mit dieser Konfiguration direkt mit dem Fahrzeug über den Fahrzeughersteller kommunizieren.

Die `vehicles` Konfiguration ist eine Liste von verschiedenen Fahrzeugen.

**Beispiel**:

```yaml
vehicles:
  - name: Zoe
    type: ...
    ...
```

Konfigurationen für bekannte Fahrzeuge sind unter [Geräte - Fahrzeuge](/docs/devices/vehicles) zu finden.

Im folgenden werden nun alle möglichen Parameter erklärt.

## Name

:::caution Erforderlicher Parameter
:::

**`name`**: Eine Kurzbezeichnung des hier konfigurierten Fahrzeigs. Der Wert wird in der Referenzierung des Fahrzeugs in der Konfiguration des [Ladepunktes](loadpoints#vehicle--vehicles) verwendet.

**Beispiel**:

```yaml
  name: Zoe
```

## Type

:::caution Erforderlicher Parameter
:::

**`type`**: Dies ist der evcc spezifische Messgeräte Typ, mit Hilfe dessen mit dem Gerät kommuniziert werden kann. Den passenden Typ für bekannte Geräte findet man unter [Geräte - Fahrzeuge](/docs/devices/vehicles).

**Beispiel**:

```yaml
  type: renault
```

Im folgenden sind die verschiedenen möglichen Typen dokumentiert:

_`audi`_: Fahrzeuge von Audi, z.b. eTron, Q55

**Beispiel**:

```yaml

  ...
```
